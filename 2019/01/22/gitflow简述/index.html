git flow是什么

Git Flow是构建在Git之上的一个组织软件开发活动的模型，是在Git之上构建的一项软件开发最佳实践。Git Flow是一套使用Git进行源代码管理时的一套行为规范和简化部分Git操作的工具。

git flow中的分支

  • 主分支
    主分支是所有开发活动的核心分支，所有的开发活动产生的输出物最终都会反映到主分支的代码中。主分支分为master和development分支。

  • master分支
    master分支上存放的应该是随时可供在生产环境中部署的代码。当开发活动告一段落，产生了一份新的可供部署的代码时，master分支上的代码会被更新。同时，每一次的更新，最好添加对应的版本号标签，也就是TAG。
